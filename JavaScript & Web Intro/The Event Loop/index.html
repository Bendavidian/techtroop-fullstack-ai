The Event Loop

Understand how JavaScript handles code execution, asynchronous tasks, and timing.

1. Call Stack

The Call Stack is where JavaScript keeps track of which function is currently running. Functions are pushed onto the stack when called and popped off when they return. JavaScript is single-threaded — only one thing runs at a time.

2. Blocking Code

Blocking code prevents anything else from running until it finishes. A long-running synchronous task (like a heavy loop) freezes the browser — no clicks, no animations, nothing responds.

3. Async & setTimeout

setTimeout is a Web API provided by the browser. JavaScript hands the timer off to the browser and keeps running. When the timer fires, the callback is placed in the Callback Queue and waits for the Call Stack to be empty before executing.

4. The Event Loop

The Event Loop continuously checks: "Is the Call Stack empty? Is there anything waiting in the Callback Queue?" If yes to both, it moves the next callback onto the stack to run. This is how JavaScript achieves non-blocking async behaviour despite being single-threaded.
